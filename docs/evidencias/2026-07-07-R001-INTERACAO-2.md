## Objetivo

Implementar login JWT, refresh token rotativo, logout, recuperação de senha e verificação de e-mail.

## Escopo entregue (conforme seção 23 do documento R001)

### Entidades
- `RefreshToken` — token rotativo com hash, expiração, revogação
- `PasswordResetToken` — token de reset com hash, expiração, uso único

### Endpoints
| Método | Rota | Descrição |
|--------|------|-----------|
| POST | `/api/auth/login` | Autentica com email+senha, retorna JWT + refresh token + permissões |
| POST | `/api/auth/logout` | Revoga o refresh token atual |
| POST | `/api/auth/refresh-token` | Rotaciona refresh token, gera novo JWT |
| POST | `/api/auth/forgot-password` | Gera token de reset (sem SMTP, apenas cria) |
| POST | `/api/auth/reset-password` | Redefine senha com token válido |

### Regras de segurança
- Bloqueio por status: user disabled/locked, organization suspended/canceled
- Refresh token com rotação (old revoked on use)
- Senhas hasheadas com BCrypt
- Tokens armazenados como SHA256 hash
- JWT com claims: userId, email, name, organization_id, role_id
- Validação de issuer, audience, signing key

### Configuração JWT
- Secret key em `appsettings.json` (64 chars)
- Access token: 15 minutos
- Refresh token: 7 dias
- Autenticação via `Bearer` scheme
