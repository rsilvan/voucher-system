## Objetivo

Implementar a base de identidade e organização do Voucher System, incluindo entidades, migrations, seed de roles/permissões, endpoint self-service de cadastro e testes unitários.

## Baseline antes da interação

- Código-fonte inexistente (codebase vazia)
- PostgreSQL rodando em Docker com database `voucherify`
- Redis rodando em Docker
- .NET 10 SDK e Node 22 disponíveis

## Escopo entregue (conforme seção 23 do documento R001)

### Entidades de domínio
- `Organization` — conta principal do cliente
- `User` — usuário humano (login, senha, status)
- `OrganizationMember` — vínculo User + Organization + Role
- `Project` — contexto operacional dentro da organização
- `Role` — papel com permissões (system/custom)
- `Permission` — ação específica em recurso
- `RolePermission` — associação N:N
- `Plan` — plano da organização

### Contracts/DTOs
- `CreateOrganizationRequest` — payload do cadastro
- `CreateOrganizationResponse` — resposta do cadastro
- `ErrorResponse` — erro padronizado

### Application layer
- `IOrganizationService` / `OrganizationService` — lógica de criação
- `IOrganizationRepository` — abstração de persitência
- Validações: campos obrigatórios, email único, senha forte, termos aceitos

### Infrastructure layer
- `VoucherSystemDbContext` — EF Core 10 + PostgreSQL + mappings
- `OrganizationRepository` — implementação transacional
- `SeedData` — plans (Trial, Free), 55 permissions, 3 system roles (Owner, Admin, ReadOnly) com permissões associadas

### API layer
- `POST /api/self-service/organizations` — endpoint público de cadastro
- `GET /api/health` — health check
- Swagger UI em `/swagger`
- CORS aberto para desenvolvimento

### Database
- Migration `InitialCreate` — 7 tabelas com índices e constraints
- Seed executado automaticamente na inicialização

### Testes unitários (8)
1. Criação bem-sucedida retorna response completo
2. Email duplicado → ArgumentException
3. Senhas diferentes → ArgumentException
4. Termos não aceitos → ArgumentException
5. Senha curta (< 8) → ArgumentException
6. Nome da organização vazio → ArgumentException
7. Plano Trial não encontrado → InvalidOperationException
8. Role Owner não encontrada → InvalidOperationException
