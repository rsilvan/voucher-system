## Objetivo

Implementar endpoints de convite, aceite, gestão de status de membros e proteção do último owner.

## Escopo entregue (conforme seção 23 do documento R001)

### Entidades
- `Invitation` — convite com token hash, status (Pending, Accepted, Expired, Revoked), expiração 7 dias
- `InvitationProjectAccess` — projetos associados ao convite

### Endpoints

| Método | Rota | Proteção | Descrição |
|--------|------|----------|-----------|
| GET | `/api/organizations/current/members` | JWT + users.read | Lista membros |
| GET | `/api/organizations/current/members/{id}` | JWT + users.read | Detalhe do membro |
| POST | `/api/organizations/current/members/invite` | JWT + users.invite | Convidar membro |
| POST | `/api/organizations/current/members/{id}/disable` | JWT + users.disable | Desativar membro |
| POST | `/api/organizations/current/members/{id}/enable` | JWT + users.enable | Reativar membro |
| GET | `/api/invitations/{token}` | Público | Validar token |
| POST | `/api/invitations/{token}/accept` | Público | Aceitar convite |

### Regras de negócio implementadas
- Email duplicado verificado (pending invitations + users)
- Token de convite com hash SHA256
- Convite expira em 7 dias
- Proteção do último owner (não permite desabilitar)
- Transação ao criar usuário + member no aceite
