## Objetivo

Implementar middleware de contexto do usuário, permission handler com cache Redis e proteção de endpoints por permissão.

## Escopo entregue (conforme seção 23 do documento R001)

### IPermissionCache (Application)
- Interface para cache de permissões
- `GetPermissionsAsync`, `SetPermissionsAsync`, `InvalidateRoleAsync`

### RedisPermissionCache (Infrastructure)
- Implementação via StackExchange.Redis (localhost:6379)
- Chave: `perms:{roleId}`
- Cache TTL: 30 minutos com invalidação manual

### UserContextMiddleware (API/Middleware)
- Extrai claims do JWT: UserId, OrganizationId, RoleId, MemberId
- Carrega permissões do cache Redis (fallback para DB)
- Popula `HttpContext.Items["Permissions"]` como `HashSet<string>`
- Registrado no pipeline após `UseAuthorization()`

### PermissionExtensions
- `RequirePermission("perm.key")` — extension para RouteHandlerBuilder
- Usa endpoint filter para verificar permissão antes da execução
- Retorna 403 se não autorizado
- `GetUserContext()` — helper para acessar contexto no endpoint

### Endpoint protegido de exemplo
- `GET /api/auth/me` — retorna user context (protegido com JWT + `organization.read`)
